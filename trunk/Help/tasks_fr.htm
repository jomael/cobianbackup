Tâches

Une tâche peut être créée ou modifiée à partir du menu Tâche.

Onglet Général

Nom de la tâche - Le nom de la tâche. Il sera utilisé plusieurs fois dans le fichier journal et pour les noms des fichiers compressés. Vous pouvez laisser le nom proposé par défaut ou choisir un nom unique pour chaque tâche.

ID de la tâche - Il s'agit d'un identifiant unique de chaque tâche. Vous ne pouvez pas le modifier.  l'ID est seulement utilisé en interne par le programme.

Désactiver - (décochée par défaut) Si cette option est cochée, la tâche sera ignorée lors de la sauvegarde.

Inclure les sous-répertoires - (cochée par défaut) Si cette option est cochée, lors de la copie de répertoire, cela copiera récursivement tous les fichiers des sous-répertoires.

Créer des sauvegardes séparées avec date/heure - (cochée par défaut) Cela créera un nouveau fichier ou répertoire à chaque copie. La date et l'heure seront ajoutées au nom de fichier. Exemple : fichier1 2002-01-11 ; 12;00.zip. Dans le cas contraire la sauvegarde sera écrasée Il y a un désavantage à écraser les sauvegardes : Si la sauvegarde échoue au milieu de la copie quelque soit la raison, vous aurez au final une sauvegarde erronée , car la sauvegarde précédente sera remplacée par la nouvelle erronée.

Purger l'attribut archive: Le programme utilise le bit archive d'un fichier pour savoir si le fichier doit être copié lors d'une sauvegarde différentielle ou incrémentielle. Si cette case est décocheé les attributs ne seront pas effacés. C'est ce qu'il faut faire si vous utilisez les mêmes sources dans plusieurs tâches.

Utilisez les attributs de fichiers : Parfois, vous ne souhaitez pas utiliser cette méthode (par exemple, si la source est une partition Linux). Dans ce cas, vous devez décocher cette option. Le programme comparera la date et l'heure du fichier source avec la destination dans le cas des copies incrémentielle et différentielle. C'est une solution très LIMITE : cela ne marche pas avec les fichiers compressés et les fichiers par FTP.

%PROGRAMNAME% supporte 4 types de sauvegardes :

Complète : Chaque fichier de la source sera copié ou compressé. Si vous avez coché "écraser les sauvegardes existantes", tous les fichiers seront remplacés. Sinon vous aurez plusieurs copies de la même source.

Incrémentielle -  Le programme vérifiera si la source est différente depuis la dernière sauvegarde. Si ce n'est pas le cas, il n'aura pas besoin de copier le fichier, il sera ignoré, ce qui permet d'éviter de le recopier. La procédure incrémentielle vérifie le bit archive des attributs du fichier. Vous pouvez manuellement réinitialiser ce dernier pour forcer une sauvegarde complète : Utiliser la fonction "Réinitialiser les attributs d'archives" du menu Tâche.

Différentielle : Le programme vérifiera si la source est différente depuis la dernière sauvegarde COMPLETE. Comme pour l'incrémentielle, il ne copiera pas le fichier si la source n'a pas été modifiée. Le fichier sera donc ignoré et la copie sera plus rapide. La procédure Différentielle vérifie le bit archive des attributs du fichier. Vous pouvez manuellement réinitialiser ce dernier pour forcer une sauvegarde complète : Utiliser la fonction "Réinitialiser les attributs d'archives" du menu Tâche.

Sauvegarde factice : Cette sauvegarde n'a pas besoin de source ou de destination. Elle sert uniquement à planifier un évènement comme exécuter une application, fermer un service, redémarrer l'ordinateur, etc.

Copies complètes à garder : Si vos sauvegardes ne sont PAS écrasées (définie par l'option "écraser les sauvegardes précédentes"), chaque nouvelle sauvegarde créera un nouveau fichier ou répertoire du même nom, la date et l'heure seront ajoutées au nom.

Par exemple: Si le fichier original est SimCity.exe, et que les sauvegardes ne sont pas écrasées, les nouvelles sauvegardes créées seront :

SimCity.exe
SimCity 2003-04-13 12;22.exe
SimCity 2003-04-14 12;22.exe
SimCity 2003-04-15 12;22.exe , etc

C'est maintenant facile de maintenir un nombre limité de copies complètes pour chaque sauvegarde et d'effacer les anciennes. Une des options permet de réaliser ceci (nombre de copies complètes à conserver), par exemple si vous ne souhaitez conserver que 2 copies complètes. Il faut noter que si votre tâche est incrémentielle ou différentielle, le programme de sauvegarde conservera ou effacera le contenu entier d'une sauvegarde complète et les enfants associés à cette sauvegarde. .

Les anciennes sauvegardes seront alors effacées. Vous pouvez toujours PARQUER une sauvegarde afin de la protéger contre l'effacement. C'est très important pour les sauvegardes incrémentielles quand la première sauvegarde est complète et que la suivante est partielle.

Cycle des sauvegardes complètes:  Si vous avez planifiez une sauvegarde différentielle ou incrémentielle alors le programme, toutes les X sauvegardes fera une sauvegarde complète. Par exemple, si vous entrez le nombre 3, les sauvegardes 3, 6, 9, 12, etc..., seront complètes. Cette fonctionnalité fonctionne seulement si vous n'écrasez pas les sauvegardes.

Onglet Fichiers

Source - Entrer les fichiers, répertoires et/ou les sources FTP à sauvegarder. Noter qu'une sauvegarde VENANT d'un serveur FTP doit être complète. Les autres types de sauvegardes seront ignorés et la sauvegarde sera réalisée comme sauvegarde complète. Vous devez avoir assez d'espace disponible dans votre répertoire temporaire pour le téléchargement complet de votre source FTP.

Destination - Entrer la destination locale et/ou le répertoire FTP. Vous pouvez avoir plusieurs répertoires. Une copie sera placée dans chaque répertoire de destination.

Onglet Fichiers

Source - Entrer les fichiers, répertoires et/ou les sources FTP à sauvegarder. Noter qu'une sauvegarde VENANT d'un serveur FTP doit être complète. Les autres types de sauvegardes seront ignorés et la sauvegarde sera réalisée comme sauvegarde complète. Vous devez avoir assez d'espace disponible dans votre répertoire temporaire pour le téléchargement complet de votre source FTP.

Destination - Entrer la destination locale et/ou le répertoire FTP. Vous pouvez avoir plusieurs répertoires. Une copie sera placée dans chaque répertoire de destination.

Onglet Planification

Type de planification : Permet de définir quand les tâches seront exécutées.

Une fois : Basiquement, les copies du fichier ou du répertoire seront réalisées à la date et l'heure que vous définissez.

Quotidienne : Réalisée chaque jour à l'heure spécifié.

Hebdomadaire : Réalisée chaque semaine au(x) jour(s) et aux heures spécifié(s).

Mensuelle : Réalisée une fois par mois, au jour et heure spécifiés

Minuterie : Réalisée toutes les X minutes

Manuellement : Réalisée uniquement si l'utilisateur clique sur "Lancer les sauvegardes". Non planifiée.

Onglet Archive

Compression : Choix de la méthode de compression à utiliser (aucune, Zip ou SQX)

ATTENTION - Si vous écrasez les sauvegardes existantes et que vous utilisez la compression, vous obtiendrez un effet de mirroir : Si un fichier est supprimé de la source, il sera AUSSI inexistant dans la sauvegarde compressé. Cela n'arrive pas if la sauvegarde n'est pas compressée ou si vous n'écrasez les sauvegardes.

Options de découpage : Si cette option est cochée, l'archive compressée finale sera découpée en morceaux. Très pratique si vous souhaitez copier les parties du fichier sur un support amovible.

Commentaire: Commentaire à inclure dans l'archive

Cryptage avancée : Choix du cryptage à appliquer

Pour plus d'information à ce sujet cliquez ici.

Onglet Spécial

Vous pouvez ajouter ici une liste de fichiers/répertoires à exclure ou inclure pour la sauvegarde. Vous pouvez également utiliser des masques : Exemple *.txt

Voici ce que vous pouvez faire :

1) Fichiers. Sélectionnez juste un fichier avec "Ajouter fichier""; button

2) Répertoires : Sélecitonnez un répertoire en cliquant sur "Ajouter répertoire"

3) Entrez un masque. Vous pouvez utilisez des masques simples comme *.txt pour exclure ou inclure que les fichiers de type texte ou vous pouvez utiliser des masques complexes comme

*\répertoire\*\*.txt

Cela inclura ou exclura tous les fichiers de type texte dans toute l'arborescence qui contient un répertoire appelé 'répertoire'.

Onglet Evènements

Avant/Après la sauvegarde : Vous pouvez spécifier des évènements à réaliser avant que la sauvegarde ne démarre et des évènements une fois que la sauvegarde sera terminée. Cela permettra de sauvegarder des fichiers utilisés par une autre application. Avant la sauvegarde, Fermer l'autre application et après la sauvegarde, relancer la. Vous pouvez arrêter/démarrer des services également.

Si vous utilisez l'évènement "Executer" or "Executer et attendre" vous pouvez passer en paramètres la source ou la destination. Pour faire cela :

* Selectionnez un évènement et un exécutable à lancer.
* Un fenêtre s'ouvrira et vous demandera des paramètres. Ici vous pouvez saisir des paramètres, comme un nom de fichier, etc...

Les fichiers/répertoires doivent être passés en utilisant des guillemets pour eviter des problèmes avec les noms de fichiers longs.

Vous devez posséder les droits d'administrateurs pour fermer des programmes ou démarrer/arrêter des services.

Onglet avancé

Lancer la sauvegarde sous un autre compte utilisateur : Si vous avez besoin d'une tâche qui nécessite d'accéder à des ressources pour lesquelles vous n'avez pas les droits sélectionnez cette option et entrez le login utilisateur et son mot de passe. Quand la sauvegarde commencera, le programme essaiera de se connecter sous ce compte et reviendra sous l'ancien à la fin de cette tâche.

Annuler en cas d'erreur: Si le changement d'utilisateur échoue, alors abandonner la tâche.

ID: nom du compte

Domaine: Nom du Domaine à placer devant le nom d'utilisateur. Entre un point s'il s'agit d'un compte local.

Mot de passe : Le mot de passe du compte.
