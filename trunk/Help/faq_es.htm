Q: Estoy tratando de ejecutar una tarea usando otra cuenta de usuario, pero esto resulta en un error. Por qué?

Para que esta función se ejecute correctamente , Vd. necesita Windows NT 4.0 SP6 o una versión mas moderna (2000/XP,2003, etc). En Windows 2000, la cuenta bajo la que se está ejecutando el servicio o aplicación debe tener el privilegio especial SE_TCB_NAME, lo que significa que en Windows 2000 esta función se ejecutará correctamente solo cuando el servicio está ejecutándose bajo la cuenta de Local System.

Q: Quiero cerrar un programa. ¿Cuál es la "titulo de ventana ventana" que tengo que introducir?

A: La orden de Windows es el lugar que contiene el título de la ventana :

En este caso (dibujo) puedo introducir el título "Dokument1-Microsoft Word" o parte del título "Dokument1" o "Microsoft Word". Note que debe introducir las palabras exactamente como se encuentran el la ventana (PROCESO SENSIBLE).

Algunos programas cambian el título de la ventana dependiendo del documento que están usando, por ello trate de usar una palabra que sepa que estará presente en el título de la ventana cuando se vaya ha cerrar el programa. Otro problema: puede haber múltiples ventanas usando el mismo título. En este caso, solo una será cerrada. Intente usar el título exacto como parámetro para cerrar programas, pero también puede usar solo una parte del título si está seguro que solo una ventana contiene esas palabras.

En esta versión se puede también introducir el nombre de la clase de la ventana que desea cerrar. Si Vd. es un usuario avanzado puede utilizar alguna herramienta como Spy++ para determinar el nombre de una clase. Para utilizar el nombre de una clase deberá usar la sintaxis:

class:nombre_de_la_clase

incluyendo la palabra clave "class:" con los dos puntos.

 

Q: Un mensaje que dice que el motor no se ha encontrado aparece en la interfaz de usuario

Existen diferentes posibilidades. La interfaz de usuario es el programa que el usuario utiliza para manejar las listas y las tareas. Ésta comunica con el motor (que se ejecuta como servicio o como aplicación) a través del protocolo TCP/IP.

1) Compruebe que el servicio está ejecutándose (si el programa se instaló como servicio)

2) Si el programa se instaló como aplicación, ejecute de nuevo el programa desde el botón de Inicio, o ejecute el programa Cobian.exe.

Otro problema podría ser que el servicio no pudo ser instalado por haber introducido un nombre de usuario o contraseña equivocado. Cambie la contraseña o el nombre de usuario en el panel de control o reinstale la aplicación. También puede utilizar el diálogo de Opciones para cambiar dichos datos.

Q: Mi cortafuegos dice que %PROGRAMNAME% quiere acceder a la internet. Por qué?

%PROGRAMNAME% solo necesita acceso a la Internet para la función de renovación automática. Desactive esta función si esta no es de su agrado.

Q-Como puedo iniciar el respaldo de todas las tareas cuando Windows se esta iniciando? Y cuando Windows está terminando?

Si Vd. está usando la función de autoinicio del programa puede abrir el Registro en la llave:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

o

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

Encuentre el valor "%PROGRAMNAME%" o "%PROGRAMNAME%user interface". Edite el valor hallado adicionando -bu después del nombre del programa.

No es posible iniciar un respaldo cuando Windows está terminando, puesto que el sistema detendría el servicio en el medio del trabajo.

Q: ¿Que argumentos puedo usar con %PROGRAMNAME% en línea de comandos? 

A:

En esta versión de %PROGRAMNAME% no existe una utilidad separada para la línea de comando. Vd. puede usar el programa principal Cobian.exe desde la línea de comandos. Los argumentos son:

-nogui  No mostrar la interfaz de usuario

-autoclose Cerrar el programa cuando el respaldo termine

-bu Respaldar todo inmediatamente

-list:(NombreDeLista) Arrancar el programa con la lista especificada en NombreDeLista. Si el nombre de la lista contiene espacios, incluir TODO el argumento entre comillas.

-m: Ejecuta el programa con la interfaz de usuario abierta. La interfaz no usará el icono de bandeja (tray icon)

Ejemplo: Cobian.exe "-list:C:\Mi documento\List.lst" -bu -nogui

 

Q: Instalando el programa como servicio, obtengo un error en el archivo log. El programa no puede acceder al disco de red. ¿Por que?

A: Para poder acceder a los recursos de red, el servicio no puede ejecutarse en una cuenta local del sistema:

Una manera de arreglar el problema:

Abra el diálogo de Opciones y cambie los datos de Logon.

Otra manera mas complicada:

* Desinstale el programa.

* Vuelva a instalarlo otra vez, pero selecciones la opción "Usar la siguiente cuenta".

* Entonces podrá introducir una identificación de usuario y una contraseña. Esta cuenta debe tener permiso para acceder al disco de red.

Q: OK, he instalado el programa como servicio, he seleccionado la opción anterior y he introducido una identificación y una contraseña válidos, pero el servicio no funciona. ¿Por que?

A: Primero asegúrese que ha introducido la identificación y contraseña correctas, recuerde que las contraseñas en sistemas NT son un caso delicado. Si está usando una cuenta de dominio, introduzca el nombre de usuario usando el formato

DOMINIO/SuIDENTIFICACIÓN.

En esta versión del instalador, si escoge acceder como un usuario existente, el instalador entenderá que el usuario tiene los privilegios para acceder como servicio. En algunos sistemas (XP y W2000 en algunas versiones) si tiene una contraseña VACÍA , el usuario no podrá acceder como servicio. Tiene 2 opciones:

1) Use una cuenta sin contraseña vacía o cree una contraseña para esta cuenta.

2) Cambie la política de su ordenador para permitir a contraseñas vacías acceder como servicio. En Windows XP/2000 encontrará esta opción en el Panel de control-Herramientas administrativas-Directiva de seguridad local.

Si piensa que ha introducido mal el nombre de usuario o la contraseña entonces:

* Abra el Panel de control

* Si está en NT, pinche Servicios, en XP/2000 pinche en Herramientas administrativas-Servicios.

* Localice el servicio llamado %PROGRAMNAME%.

* Haga doble clic en el o pinche el botón "Propiedades".

* Seleccione la pestaña "Iniciar sesión" (puede haber otras opciones dependiendo de la versión del SO).

* Vuelva ha introducir el usuario y la contraseña.

* Pinche Aceptar.

* Inicie el servicio o reinicie el ordenador.

Q: ¿Por que el programa falla respaldando algunos archivos?

A: Windows y otros programas pueden abrir algunos archivos con acceso exclusivo. En este caso el programa de respaldo (y cualquier otro programa) fallarán al intentar copiar ese archivo. Solución : Cierre el programa que está usando el archivo que esta intentando respaldar.

Q: ¿Existe alguna función de recuperación?

A: %PROGRAMNAME% no utiliza ningún formato propietario de respaldo por lo que una función de recuperación es totalmente innecesaria.

Q: ¿Por que el programa falla al comprimir? Tengo suficiente espacio libre en el disco de destino.

A: La operación de comprimir usa el directorio temporal para comprimir. Si no tiene suficiente espacio en el directorio temporal, cuando se comprima un gran archivo o una gran cantidad de datos, el archivo zip puede resultar corrupto. Cambie su directorio temporal para comprimir en el diálogo de Opciones.

A: Esta información se aplica a la versión 7.3 o anteriores. A partir de la versión 7.4 el programa puede ejecutar aplicaciones en el contexto del usuario.

Está usando el programa como servicio. Un servicio funciona en el contexto de otro escritorio. El programa que inicio ESTA FUNCIONANDO  pero es invisible para usted, porque fue iniciado  por un servicio.  Puede fijar el servicio para que funcione interactivamente con el escritorio, pero esto no es recomendable (este servicio no fue diseñado con ese para ese propósito. Hágalo bajo su responsabilidad). También puede usar el programa como aplicación para solventar este problema.

Q: Quiero que su programa cree un espejo por mi. Cuando borre el archivo original, quiero que el programa borre ese archivo del respaldo.

Que sucedería si un virus destruye sus ficheros y cuando quiera recuperarlos se encuentra conque %PROGRAMNAME% ha borrado los mismos ficheros, pues Ud.le ordenó hacer un espejo?

Q: Estoy intentando respaldar a un CD-RW, pero obtengo un error que dice "No se encontró el directorio de destino. "Por que?

A: Este programa no incluye ningún driver para usar con un grabador de CD, por esto no puede respaldar directamente en un CD-RW. Haga el respaldo en su disco duro y luego grábelo manualmente un CD usando su programa de grabación favorito. Hay programas de terceras partes como as Adaptec Direct CD (Escritura Directa) que pueden hacer posible que %PROGRAMNAME% interactue directamente con su grabador de CD. Si usted tienes este u otro software similar instalado, puede respaldar directamente en su grabadora. Tenga en cuenta que %PROGRAMNAME% es un programa multitarea y por ello múltiples tareas pueden acceder a su CD al mismo tiempo. Es más, cuando se comprime, toda la operación de comprimir se lleva acabo en el directorio de destino y su CD puede hacerse ilegible o corrupto. La frase final es: NO respalde en un CD-RW directamente.

Q: He creado un evento "Post-respaldo" para cerrar Outlook, pero éste no se cierra o genera una excepción. ¿Por qué?

Outlook  no es una aplicación normal. Este programa trabaja de una forma diferente a todos los demás. A veces ignora los mensajes que le piden que se cierre, a veces cierra generando una  excepción, a veces cierra normalmente. Resumiendo, Outlook no puede ser cerrado usando este método.

Q: Puedo hacer un respaldo directo a CD o DVD?

%PROGRAMNAME% no contiene controladores para grabadoras de CD-DVD.. No es recomendable hacer un respaldo directo a un CD. Ud puede usar alguna utilidad que permita copiar ficheros directos al CD, comop por ejemplo DirectWrite. Nunca use compresión cuando está respaldando a un CD o DVD. La compresión nunca ocurre de forma secuencial, sino que tiene que crear muchos ficheros temporales, etc.  El método normal de compresión también funciona mal en este caso. Si Vd. desea tanto hacer una copia directa, marque "Usar siempre ShFileOperation".

Q - ¿Por qué %PROGRAMNAME% no trabaja con FastSwitch (cambio de usuario rápido) en Windows XP?

A- El programa puede funcionar en modo FastSwitch, siempre que ningún otro usuario presente en el sistema esté ejecutando el programa al mismo tiempo.

Q- Mi respaldo incremental o diferencial no trabaja correctamente. La tarea se ejecuta completamente, respaldando todos los ficheros. ¿Por que?

A- %PROGRAMNAME% usa el bit de archivo del atributo de los ficheros para comprobar si éste necesita ser copiado o no. A veces el programa no puede cambiar el atributo y el fichero será copiado de nuevo sin ser realmente necesario. Esto puede suceder , por ejemplo, si la fuente se encuentra en una máquina de tipo Linux u otra que tenga una forma de atributos diferentes, o si no existen los permisos necesarios para realizar esta operación.

Q- Yo puedo cambiar el bit de archivo manualmente, sin embargo, el programa falla al hacerlo. ¿Por qué?

Esto es un problema con los permisos. Recuerde que el servicio puede estar ejecutándose bajo otra cuenta diferente a la suya, (por ejemplo LocalSystem). Cambie el nombre del usuario bajo el cual se ejecuta el servicio.

 

Q- Yo quiero excluir algunos subdirectorios con el mismo nombre de muchas carpetas. Por ejemplo, tengo un subdirectorio llamado Versiones en cada carpeta. Como hago?

A - Introduzca una máscara de tipo: Versiones\*.*

Q- Estoy respaldando una tarea y todos los ficheros se han copiado. Sin embargo, en el diario hay pone que se han encontrado errores. Por qué?

Algunes errores no son críticos. Por ejemplo, tratando de hacer un respaldo a un directorio remoto que no exista, nos dará el error "El directorio remoto x no existe". Sin embargo el programa creará automáticamente el directorio y el respaldo continuará sin problemas. Este error no es crítico, pero se mostrará al final para llamar su atención. Si quiere ver todos los errores, active la opción "verbosa", pero el fichero de diario podrá ser entonces ENORME!

Q- Puede Cobián Backup respaldar ficheros con nombres en chino, japonés u otro idioma de múltiples bytes?

No. Cobián Backup solo trabaja con idiomas de un byte por caracter. En un futuro, existen planes de una versión unicode del programa.

Q- Que planes hay para la version 8?

La versión 7 será la última versión que se puede ejecutar en Windows 9x. La próxima versión será solamente soportada en Windows NT-2000-XP-2003, etc. La versión 8 será unicode.

Q- El programa se nota muy lento cuando la interfaz está mostrándose y está utilizando todos los recursos de mi máquina.

Esto sucede cuando los ficheros fuentes son demasiados (miles de ellos) o cuando estos se encuentran en un disco en la red. El programa está tratando de contar los ficheros y calculando el tamaño total de la fuente. Desactive la opción:

Opciones- Interfaz- Mostrar tamaño de ficheros

Q- Cuando instalo el programa se muestra el error: No se puede descomprimir los ficheros, o No se puede extraer el recurso.

Cosas a comprobar:

1- Que se disponga de suficiente espacio libre en el directorio temporal (50 Mb de espacio libre serán suficientes)

2- Que el usuario que está ejecutando el programa tenga los permisos necesarios para crear subdirectorios y escribir al directorio temporal.

Q- Quiero hacer un respaldo incremental o diferencial, pero el programa sigue haciendo respaldos completos. ¿Por qué?

El programa usa el bit de archivo del atributo del fichero para determinar si el fichero ha sido modificado y es necesario copiarlo. Si el bit de archivo no ha podido ser modificado por alguna razón, el programa seguirá copiando este fichero en respaldo sucesivos incluso si éste fichero no ha sido modificado. Algunas de las causas de que el bet de archivo no pueda ser modificado son:

1- El usuario del programa no tiene el permiso suficiente para modificar los atributos del fichero fuente
2- La fuente de la tarea está localizada en una partición que no comprende el sistema de atributos de Windows (Linux, MacOS, etc)

Q- Tengo 2 tareas que deben iniciarse a la misma hora, pero el segundo nunca es iniciado.

Ambos temas deben iniciarse, pero en muy raras ocasiones, procesar la lista puede demorar un poco, por lo que al llegar el turno de la segunda tarea, puede haber pasado ya el minuto exacto del respaldo, por lo que ésta no será iniciada. Para estar seguros, ponga la segunda tarea a iniciarse un minuto después de la primera.

Q- Estoy detrás de un sistema NAT. ¿Qué debo hacer para poner a funcionar mi respaldo de FTP?

Use transferencias pasivas, introduzca la dirección de IP externa en la página "Avanzado" del diálogo de FTP. Si es posible seleccione también NAT fast track.

Q- Yo quiero hacer un respaldo a una dirección FTP. He introducido la dirección ftp://miservidor.com pero recibo un error de timeout. Yo sé que ésa dirección existe.

Tu dirección es miservidor.com. No escribas el nombre del protocolo ftp://

Q- Tratando de cargar/descargar un fichero grande (>2Gb) a un FTP, el programa me de el error "Range check error"

Las librerías de FTP que estoy utilizandon tienen este límite de momento. Ficheros de mas de 2Gb no pueden ser enviados. En cuanto las librerías sean renovadas y este problema sea superado, incorporaré dichos cambios al programa para superar este límite. Mientras tanto Vd. podría comprimir el fichero usando "splitting" para dividirlo en varias partes más pequeñas.

Q- Necesito respaldar algunos ficheros que están abiertos exclusivamente por otra aplicación. Es esto posible?

Normalmente, si una aplicación mantiene algún fichero abierto exclusivamente, este no podrá ser respaldado. Sin embargo, podría probar con una de estas soluciones:

1) Use un evente "Pre-respaldo" para cerrar el programa que mantiene el fichero abierto. Use un evento "Post-respaldo" para ejecutar  el programa nuevamente.

2) Microsoft ha publicado un script que permite cerrar todos los ficheros en uso. Este script puede ser ejecutado como un fichero bat en un evento Pre-respaldo. Esta solución funciona solamente con sistemas basados en NT.
http://support.microsoft.com/kb/290585/es

Q- Existe algún límite para el tamaño de archivos que puede ser cifrado usando métodos fuertes.

No existe un límite si está usando la versión 7.1 o posterior.

Q- Por favor, por favor, por favor implementa la función de "espejos" (mirror).

Q: Puedo ejecutar el programa sin el icono de sistema (tray icon)?

Si el programa está ejecutándose como aplicación, use el parámetro -nogui.


Si el programa está ejecutandose como un servicio, simplemente borre el valor "%PROGRAMNAME% Interface 8" de la llave de registro:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run

Q: He obtenido un fichero Zip que no puede ser abierto con WinRAR u otro programa. El fichero es mayor de 2 Gb. Por qué?

Cuando el archivo es mayor de 2Gb, el formato Zip64 es utilizado. Solo algunos pocos programas pueden entender este formato. WinZip 9, ZipGenious o el programa descompresor incluido en %PROGRAMNAME% son algunos de ellos.

Q: Cómo puedo excluir el directorio de Cache del Firefox?

Simplemente introduzca las siguientes máscaras en la lista de ficheros a excluir:

XUL.mfl
Cache\_CACHE*
Cache\*d01
parent.lock

Q: Por qué utilizas el método de atributos para respaldos incrementales y diferenciales y no el método de comparación de fechas como en versiones anteriores?

Por muchas razones, pero las principales son que, usando el método de fechas no sería posible hacer respaldos incrementales o diferenciales sin re-escribir los respaldos existentes. Al no existir el fichero de destino, las fechas no podrán ser comparadas. Además, también sería muy difícil efectuar respaldos no-completos a través de FTP, puesto que el formato de fecha varía en dependencia del tipo de sistema operativo en el que se encuentre el servidor FTP.

Q: Mi servicio ha dejado de trabajar. Por qué?

Si su servicio está ejecutándose bajo una cuenta diferente a LocalService, y la contraseña de esta cuenta es cambiada, el servicio tratará de comenzar la sesión usando la contraseña vieja, y fallará. Abra el Panel de Control - Herramientas de Administración - Servicios. Seleccione el servicio del programa y deténgalo. Cambie la contraseña y reinicie el servicio a continuación.

Q: Cuando estoy cargando o descargando desde un FTP, mi servicio se "cuelga".

Esto ocurre si su ordenador se encuentra en una red de tipo NAT. Trate de experimentar con todas las opciones del diálogo de FTP (IP externa, Pasivo, etc). En cuanto se encuentre una solución estable al problema, esta solución será publicada en un parche.

Q: El programa funciona perfectamente cuando se está ejecutando como administrador en XP/2000, pero no funciona cuando este se ejecuta como un usuario normal.

Controle los permisos de sistema NTFS para el directorio en el cual el programa está instalado. Déle permiso de lectura a todos los usuarios. Especialmente importante es el fichero engine.ini y el subdirectorio DB. Déle permiso de escritura y lectura a ambos, para todos los usuarios.

Q: AddAware reporta que libeay32.dll and ssleay32.dll son  spyware. (programas espías)

A: Sí, como no, y mi abuela es una bicicleta. Estas dos bibliotecas son parte de OpenSSL usado para transferir ficheros a servidores de FTP seguros (usando SSL). Si Vd. no usa SSL y esto le hace sentir mejor, borre ambos ficheros.
