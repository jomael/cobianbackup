Application contre Service

Un service est une application spéciale de Windows qui s'exécute même si aucun utilisateur n'est connecté au sytème. Le service peut tourner avec un compte spécial (compte Système) ou avec un autre compte existant. Sous Windows NT, 2000 ,XP ou 2003 il est recommandé d'exécuter le programme en tant que service. Le choix de faire tourner le programme en tant que service ou application se fait lors de l'installation du programme. Vous pouvez changer le type d'installation depuis les Options dans le menu "outils".

Quand %PROGRAMNAME% tourne en tant que service, un problème peut arriver : si vous souhaitez accéder à un disque ou répertoire partagé ou encore à une ressource non-locale, vous ne pouvez pas faire tourner le service avec le compte système par défaut. Vous devez le faire tourner avec un compte qui peut accéder au réseau (exemple: le compte administrateur). Lors de l'installation, vous pouvez choisir de configurer le service pour le faire fonctionner avec le compte système ou un autre.

Une autre différence est que vous ne pouvez pas fermer un service comme vous fermez une application. Vous arrêtez/démarrez un service en utilisant le panneau de configuration - outils d'administration - icône Services. Vous pouvez aussi le contrôler depuis les Options dans le menu "outils".
